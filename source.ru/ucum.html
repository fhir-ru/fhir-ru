Применение UCUM в FHIR

Кодовая система Единицы измерений (UCUM) рекомендуется для использования в типе данных Quantity.

Сводные данные

SourceUCUM создан под управлением Regenstrief Institute, Inc и The UCUM Organization по адресу http://unitsofmeasure.org
SystemUCUM-коды идентифицирует URI http://unitsofmeasure.org
VersionЕсли используется версия, она должна быть стандартной версией UCUM, например 1.9. Нет необходимости указывать версию в типе данных Coding, только в наборах значений, использующих коды UCUM
CodeДопустимые выражения с использованием чувствительных к регистру обозначений
DisplayЭлемент display не определен; для отображения используется непосредственно сам UCUM-код
Filter PropertiesОпределены два свойства фильтра, приведенные ниже

Свойства UCUM-фильтра

В данном разделе задокументированы фильтры по свойствам, которые можно использовать в кодовой системе UCUM в запросах на компоновку наборов значений.

Фильтр по свойствам

ОписаниеОграничивает выражение конкретным базовым свойством UCUM
Имя свойстваproperty
Допустимые операции=
Допустимые значения[строковое имя свойства]
КомментарииОграничивает выражения до некоторого выражения, которое сравнимо с базовой единицей измерения с совпадающим значением свойства. Примечание: оно не совпадает со свойством в единицах, только в базовых единицах

Канонический фильтр

ОписаниеУчитывает любое выражение, которое сопоставимо с названной единицей измерения
Имя свойстваcanonical
Допустимые операции= / in
Допустимые значенияUCUM-выражение
КомментарииУчитывает любое выражение, которое сопоставимо с данной единицей(-ами) измерения

Имплицитные наборы значений

Имплицитные наборы значений - те, спецификация которых может быть предсказана на основе грамматики лежащей в основе кодовой системы и известной структуры URL, который ссылается на них. В UCUM определена одна группа имплицитных наборов значений: по каноническому значению (By Canonical Value).

Если какие-либо ресурсы value set созданы с идентификатором, который соответствует одному из шаблонов URL, приведенных ниже, содержимое таких ресурсов должно соответствовать приведенному шаблону. Профилям и другим ссылкам на наборы значений допускается ссылаться на эти наборы значений напрямую.

Идентификатор набора значений http://unitsofmeasure.org/vs - это набор значений, который содержит все UCUM-коды.

Multi-Axial Hierarchy Entry

Набор значений с идентификатором "http://unitsofmeasure.org/vs/[expression]" должен соответствовать данному шаблону, где [expression] - это допустимое UCUM-выражение: 

<ValueSet xmlns="http://hl7.org/fhir">
  <text>
    <status value="generated"/>
    <div xmlns="http://www.w3.org/1999/xhtml">
      [some html that identifies that this value set 
      includes all UCUM expressions that are comparable to the provided expression]
    </div>
  </text>
  <identifier value="http://unitsofmeasure.org/vs/[expression]"/>
  <version value="[optional - but recommended - UCUM version]"/>
  <name value="Ucum Expressions comparable to [expression]"/>
  <description value="Ucum Expressions comparable to [expression]"/>
  <status value="active"/>
  <date value="[optional date of LOINC release]"/>
  <compose>
    <include>
      <system value="http://unitsofmeasure.org"/>
      <filter>
        <property value="comparable"/>
        <op value="="/>
        <value value="[expression]"/>
      </filter>
    </include>
  </compose>
</ValueSet>
